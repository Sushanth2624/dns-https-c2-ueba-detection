Why signatures fail

Traditional detection matches known-bad patterns. Modern C2 defeats it by design:

  • rides encrypted TLS — payload unreadable
  • rotates domains daily (DGA) — blocklists useless
  • low-and-slow beacons — blends into noise
  • tunnels through DoH — resolver can't inspect

The shift: from "is this a known-bad string?" to "does this host behave like it's talking to a controller?"

Each is a 0–1 score mapped to MITRE ATT&CK. The correlation across them is the contribution.

Glass-box correlation — the contribution

confidence = ueba_weight · anomaly + (1−w) · Σ wᵢ·behaviourᵢ + boosts
  • Transparent: every weight is inspectable and reported — no black box
  • Boosts: extra confidence when high-signal behaviours co-occur (e.g. beacon + rare TLS)
  • Explainable: we can say exactly why any host scored what it did

Limitations & future work

Limitations — honestly
  • Compressed benign baseline (minutes, not a 7-day window)
  • Container endpoints, not full VMs; lab traffic, not live enterprise scale
  • Scripted attack simulators — cleaner than real, evasive C2
  • Batch (not real-time); single-node ES; JA4 best-effort
Future work
  • Real-time streaming detection on a live network tap
  • Longer, real benign baseline; more entity diversity
  • Learned / adaptive indicator weights
  • Evaluate against real C2 frameworks (Sliver, Cobalt Strike)
  • Automated response via SOAR integration
