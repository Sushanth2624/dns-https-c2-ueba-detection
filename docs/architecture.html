01 — The core problem

Why command-and-control is so hard to see

After malware lands on a machine, it needs to phone home — to receive commands and exfiltrate data. That channel is called command-and-control (C2). Attackers deliberately hide it inside traffic every network already allows:

  • Encrypted web (HTTPS)

    The payload is inside TLS, so the packet contents are invisible. A firewall sees "a host visited a website," nothing more.

  • DNS & DNS-over-HTTPS

    DNS is required for everything, so it's rarely blocked. Malware tunnels data through it — or hides it in DoH, where even the resolver can't inspect it.

  • Looks legitimate

    Low-and-slow beacons, domains that rotate every day (DGA), traffic shaped to resemble a browser. There is no fixed "bad string" to match.

Signature and IOC matching answer "have I seen this exact bad thing before?" Against traffic that is encrypted, ever-changing, and disguised as normal, the answer is almost always no — so the attack sails straight through.

The defensive shift the security industry has made is from signatures to behaviour: not "is this a known-bad string" but "does this host behave like it is talking to a controller?" A single behaviour (say, one odd DNS lookup) is noisy and easy to dismiss. The insight of this project is that several weak behavioural signals, correlated together and baselined against normal, become a strong and defensible one.

02 — The idea

Many weak signals → one explainable verdict

The system watches every host for eight independent behaviours, asks a UEBA model how abnormal each host is versus a learned baseline of "normal," then a glass-box correlation engine fuses everything into a confidence score. Crucially, every verdict is explainable: it names which behaviours fired, why, the matching MITRE ATT&CK techniques, and what the analyst should do next — no black box.

03 — Architecture & infrastructure

How the lab is built

The environment mirrors a real enterprise segment on a single analysis machine. Endpoints are isolated hosts on a lab network; the analysis VM is their gateway and DNS resolver, so all their traffic passes through it and is captured 100% inline — every host keeps its own real IP address and TLS fingerprint. Sensors turn packets into structured telemetry; the detection engine scores it; results land in Elasticsearch and are shown in Kibana.

Because there is no nested virtualization on the host, the endpoints are lightweight containers rather than full VMs — but the network path (endpoint → inline gateway → sensors) and the per-host telemetry are exactly as a multi-VM lab would produce.

04 — Data flow

From packet to explained alert

One continuous path carries a packet all the way to a ranked, explained alert on a dashboard.

  1. Capture. tcpdump records every packet crossing the lab bridge.
  2. Sensors. Zeek writes structured logs (DNS queries, TLS SNI + JA3, connections); Suricata runs the signature baseline.
  3. Features. For each host, the eight indicators are computed as 0–1 sub-scores.
  4. UEBA. An IsolationForest + z-score model scores how anomalous the host is vs. benign.
  5. Correlate & explain. Weighted fusion + boost rules produce a confidence; if it crosses the threshold, an explainable alert is built (verdict, contributing indicators, MITRE, recommended actions).
  6. Store & visualize. Alerts, per-host scores, and raw telemetry go to Elasticsearch, rendered in three Kibana dashboards.

Why false positives stay near zero: a benign host with, say, high-entropy CDN hostnames trips one indicator — but correlation requires corroboration and a high UEBA anomaly, so it never reaches the alert threshold. Attack hosts trip several indicators at once and are baselined as clearly abnormal.

06 — Results

Behaviour + UEBA beats signatures, head-to-head

The same real capture (14 hosts, 6 benign + 8 attack) was scored three ways: A signatures only (Suricata), B the best single behavioral indicator, and C the full multi-indicator + UEBA system. Higher F1 = better overall accuracy.

C detects all four techniques (DGA, tunneling, beaconing, DoH) with zero false positives. Signatures structurally miss DGA and beaconing — there is no static string to match — so they top out at F1 0.55. Every single indicator alone misses at least one technique. Only the correlation catches everything.
